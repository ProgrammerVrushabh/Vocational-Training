# Customer Shopping Behavior Analysis

This project is a portfolio-style end-to-end customer analytics case study built around retail shopping behavior. It shows the full workflow from raw data preparation to SQL analysis, dashboarding, and presentation-ready insights.

## Project Summary

The business problem is to understand how customers shop across demographics, product categories, discount usage, shipping preferences, subscription status, and repeat-purchase behavior. The goal is to turn the dataset into practical recommendations that can improve sales, customer engagement, and long-term loyalty.

This project covers:

- Python data cleaning, preparation, and exploratory analysis
- SQL-based business analysis and segmentation
- Power BI dashboard development
- Written report and presentation assets

## Business Questions

The analysis in this project is designed to answer questions such as:

- Revenue differences between male and female customers
- How discount usage affects higher-value purchases
- Top-rated and most purchased products
- Shipping method comparisons
- Subscriber vs non-subscriber spending patterns
- Purchase behavior across age groups and customer segments

This folder contains the complete project deliverables:

## Analysis Workflow

1. Load and inspect the CSV dataset in the Jupyter notebook.
2. Clean and transform the data for downstream analysis.
3. Create or load the SQL database structure.
4. Run the business questions in `customer_behavior_sql_queries.sql`.
5. Build the Power BI dashboard from the analyzed data.
6. Summarize the findings in the PDF report and presentation.

## How To Use

1. Open `Customer_Shopping_Behavior_Analysis.ipynb` to review the Python preparation steps.
2. Run or adapt `Customer_data_base.sql` to create the database structure.
3. Execute `customer_behavior_sql_queries.sql` against your SQL database.
4. Open `customer_behavior_dashboard.pbix` in Power BI to explore the dashboard.
5. Review the report and presentation files for the final business story.
